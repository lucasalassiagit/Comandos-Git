Introduccion

¿Qué es un repositorio?

El repositorio es el área donde se guarda todo. Nos permite tener un control del cambio de versiones.

Áreas de Trabajo

1. Directorio de Trabajo (Working Directory)

Espacio donde trabajas con tus archivos locales. Aquí es donde haces cambios en los archivos del proyecto.

2. Área de Preparación (Staging Area)

Área intermedia donde preparas los archivos que deseas incluir en el próximo commit.

3. Repositorio Local (Local Repository)

Donde se almacenan todos los commits del proyecto. Contiene la historia del proyecto y las ramas.

Uso de Git

Comando Acción
$ git init Inicializar git
$ git add Arch1.txt Arch2.txt Subir archivo al area de preparacion
$ git rm --cached documento.txt Para borrar un archivo del area de preparacion
$ git status -s Informacion de archivos:
  • ??: Archivo no rastreado
  • A: Archivo agregado al area de preparacion
  • M: Modificado
  • D: Eliminado
  • D: En escritorio
  • D: En area de preparacion
$ git commit -m "Mensaje sobre commit" Subir a area de preparacion
$ git commit (se abre VSC) Subir a area de preparacion
$ git commit -a Subir desde area de trabajo a repo local

Remover archivos de un repositorio

Comando Acción
$ rm archivo.txt Remover un archivo de un repositorio
$ git add archivo.txt Agregar la eliminacion del archivo
$ git commit -m "Elimine archivo" -a Documentar eliminacion

Restore

Devuelve un archivo eliminado del area de trabajo
Comando Acción
$ git restore archivo.txt Lo devuelve desde el area de preparacion
  1. $ git restore --staged archivo.txt
  2. $ git restore archivo.txt
 		Si esta en el area de preparacion

Checkout

Restaura cambios realizados en el archivo desde el commit (no recomendado)
Comando Acción
$ git checkout archivo.txt Si no lo subi al area de preparacion

Reset

Restaura el archivo hasta el ultimo commit subido
Comando Acción
$ git reset --hard Restaura archivo desde ulitmo commit

Git diff: sirve para comparar commits

Averiguar id de commits
Comando Acción
$ git show archivo.txt Busca info en el commit y me la muestra
$ git diff --staged Comparacion entre commit y area de preparacion
$ git log Id del commit
$ git log --oneline Id del commit (muestra menos caracteres)
$ git log --all --oneline --graph Id de los commit de todas las ramas
$ git config --global core.abbrev (cant. caracteres) Configurar cant caracteres del id

Comparacion de commits
Comando Acción
$ git diff (id 1 commit) (id 2 commit) Comparacion entre commits
$ git diff --name-only (id 1 commit) (id 2 commit) Visualizar nombres de archivos que cambiaron
$ git diff --word-diff (id 1 commit) (id 2 commit) Visualizar cambios dentro de un archivo

Modificar el ultimo commit subido

PRECAUCION al usarlo en repos compartidos
Comando Acción
$ git commit --amend Modificar (en realidad se reemplaza por uno nuevo) la descripcion de un commit
  1. Creo nuevo/s archivo/s
  2. Lo subo al area de preparacion
  3. $ git commit --amend
  • Agrega la eliminacion del archivo
  • Me incluye todos los archivos con un mismo id

Deshacer un commit

Lo primero es cambiar el puntero, para que apunte al ultimo commit que quedara


UTILIZANDO SOFT (borra del commit)
Comando Acción
$ git reset --soft (id del commit) Manda los commit "eliminados" al area de preparacion
$ git reset --soft head~(numero de commit que me quiero posicionar) Para posicionar el commit que desemos detras del que esta seleccionado

UTILIZANDO MIXED (limpia el area de preparacion)
Comando Acción
$ git reset --mixed (id del commit) Los archivos los eliminara del repo y area de preparacion, solo los deja en area de trabajo

UTILIZANDO HARD (borra todos los cambios)
Comando Acción
$ git reset --hard (id del commit) Borra todos los cambios realizados en los commit posteriores al seleccionado por el puntero

Ramas

Comando Acción
$ git branch Lista de ramas y posicion

Merge ramas: fusiono ramas

Comando Acción
$ git merge rama-a-fusionar ¡¡¡DEBO ESTAR EN LA RAMA DONDE SE FUSIONARA!!!
$ git reset --hard (id de la rama antes que se fusione (master)) Para volver atras con la fusion
$ git log --oneline --all --graph (me muestra los id de los commit de todas las ramas) Para ver todos los id de commit (cuando estoy en rama master)

Merge conflicts

Suceden conflictos cuando modifico la rama master y la rama creada y las quiero combinar

Los conflicos se pueden manejar de dos formas:
  1. Los manejo desde VSC
  2. Utilizo merge editor

Para guardar los cambios:
Comando Acción
$ git merge --continue Guardo los cambios

gitignore: archivos que no se suben al commit

Es una archivo llamado .gitignore
Comando Acción
*.txt Ignora todos los archivos que son ".txt"
NombreCarpeta/ Ignora todos los archivos de dentro de una carpeta
!noIgnorar.txt No lo ignora
$ git ls-tree -r --name-only (id o HEAD) Muestra todos los archivos de un commit especifico

GIT IGNORE GLOBAL

  1. Lo utilizo para varios repositorios
  2. Lo creo en cualquier ubicacion
Comando Acción
$ git config --global core.excludesfile (ruta donde se encuentra el archivo global) Git siempre me lo agrega
